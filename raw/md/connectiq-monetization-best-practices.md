# Best Practices for Creating Monetized Content

Source: https://forums.garmin.com/developer/connect-iq/b/news-announcements/posts/tips-for-monetizing-your-apps
Fetched: 2026-06-17

## Payment Badge

If your app requires the user to pay in order to use the primary
features of the app, you should mark the app in the app detail
description as requiring payment, which will show the "Payment Required"
badge in the app store.

For secondary features (like game levels), disclosure in the description
suffices without the payment badge.

## User Experience

- Provide user guidance to help them through the payment process via
  FAQs or videos linked prominently in app descriptions
- Avoid displaying paywalls without warning
- If offering trials, communicate remaining time clearly to users
- For watch faces, use on-device settings rather than paywalls to guide
  payment flows

## Developer Guidelines

- Use real names or non-impersonating aliases
- Maintain current contact information
- Avoid claiming Garmin partnerships without written permission
- Do not manipulate reviews or ratings

Violating these standards can result in app removal.

## Trial Period

A thirty day trial period is available for trial apps. The API contains
a built-in way to unlock after purchase.

When a previously free app adds pricing, it is temporarily removed for
re-evaluation. Users must repurchase after approval.
